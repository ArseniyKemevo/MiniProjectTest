Валютный конвертер с ручным вводом курсов
Учебный проект. Приложение представляет собой консольный валютный конвертер, построенный по принципам TDD (Test-Driven Development) с разделением на слои и хранением данных в локальной базе данных SQLite.

1. Основные возможности (MVP)
Ручное управление курсами: Добавление новых и обновление существующих курсов обмена для любых пар валют.

Конвертация в реальном времени: Расчет итоговой суммы на основе введенных пользователем валют и объема средств.

История операций: Логирование всех успешных обменов с фиксацией даты, времени, курсов и сумм, а также вывод истории в удобном табличном виде.

Надежность и валидация: Защита от некорректного ввода (отрицательные суммы, неверные коды валют, отсутствие курса в системе).

2. Архитектура приложения
Проект спроектирован с разделением ответственности на три независимых слоя (3-Tier Architecture), что позволяет полностью изолировать бизнес-логику от интерфейса и покрыть её Unit-тестами.

+------------------------------------------+
|          Пользователь (Терминал)         |
+------------------------------------------+
                     │
                     ▼
+------------------------------------------+
|  Слой представления / UI (main.py)       |
|  - Текстовое меню программы              |
|  - Ввод данных и вывод результатов       |
+------------------------------------------+
                     │
                     ▼
+------------------------------------------+
|  Слой бизнес-логики (converter.py)      |
|  - Валидация валют, сумм и курсов        |
|  - Расчет конвертации                    |
+------------------------------------------+
                     │
                     ▼
+------------------------------------------+
|  Слой данных / БД (database.py)          |
|  - Инициализация таблиц SQLite           |
|  - CRUD-операции с курсами и историей    |
+------------------------------------------+
                     │
                     ▼
+------------------------------------------+
|    Файл базы данных (currency_converter.db)|
+------------------------------------------+

3. Структура данных (SQLite)
В базе данных используются две таблицы. Для обеспечения целостности данных в таблице курсов настроен составной первичный ключ с логикой ON CONFLICT DO UPDATE.

Таблица rates (Курсы валют)
Поле	Тип данных	Ограничения	Описание
from_currency	TEXT	NOT NULL, PRIMARY KEY (1/2)	Исходная валюта (например, USD)
to_currency	TEXT	NOT NULL, PRIMARY KEY (2/2)	Целевая валюта (например, RUB)
rate	REAL	NOT NULL	Курс обмена (from -> to)

Таблица history (История операций)
Поле	Тип данных	Ограничения	Описание
id	INTEGER	PRIMARY KEY AUTOINCREMENT	Уникальный номер записи
timestamp	TEXT	NOT NULL	Дата и время операции (YYYY-MM-DD HH:MM:SS)
from_currency	TEXT	NOT NULL	Код исходной валюты
to_currency	TEXT	NOT NULL	Код целевой валюты
amount_from	REAL	NOT NULL	Конвертируемая сумма
amount_to	REAL	NOT NULL	Полученная сумма
rate	REAL	NOT NULL	Курс, по которому совершен обмен

4. Инструкция по установке и запуску
Требования
Установленный интерпретатор Python 3.8 или выше.

Менеджер пакетов pip (для установки тестовых зависимостей).

Установка зависимостей
Для тестирования приложения и проверки покрытия кода тестами установите pytest и pytest-cov:
- pip install pytest pytest-cov

Запуск приложения
Запустите главный файл интерфейса из корневой директории проекта:
- python main.py

5. Запуск тестов и проверка покрытия (TDD)
Бизнес-логика приложения полностью изолирована и тестируется с использованием встроенной базы данных в оперативной памяти (:memory:), что гарантирует скорость тестов и чистоту локальных файлов.

Запуск всех тестов:
- pytest -v
Запуск тестов с генерацией отчета о покрытии кода (Coverage):

- pytest --cov=converter --cov=database tests/
Важно: Согласно критериям оценки проекта, минимально допустимый уровень покрытия кода тестами (Code Coverage) составляет 80%.
