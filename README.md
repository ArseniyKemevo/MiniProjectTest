# Валютный конвертер с ручным вводом курсов

Учебный проект по дисциплине «Технологии разработки программного обеспечения». Приложение представляет собой консольный валютный конвертер, построенный по принципам **TDD (Test-Driven Development)** с разделением на независимые слои и хранением данных в локальной базе данных **SQLite**.

---

## 1. Основные возможности (MVP)

* **Ручное управление курсами:** Добавление новых и динамическое обновление существующих курсов обмена для любых пар валют.
* **Конвертация в реальном времени:** Расчет итоговой суммы на основе введенных пользователем кодов валют и объема средств.
* **История операций:** Автоматическое логирование всех успешных обменов с фиксацией даты, точного времени, курсов и сумм, а также вывод истории в удобном табличном виде.
* **Надежность и валидация:** Полная защита от некорректного ввода (обработка отрицательных сумм, неверных форматов кодов валют, пробелов, запятых и отсутствия курса в системе).

---

## 2. Архитектура приложения

Проект спроектирован с разделением ответственности на три независимых слоя (**3-Tier Architecture**). Это позволяет полностью изолировать бизнес-логику от интерфейса и покрыть её Unit-тестами без запуска консольного меню.

```text
+------------------------------------------+
|          Пользователь (Terminal)         |
+------------------------------------------+
                     │
                     ▼
+------------------------------------------+
|  Слой представления / UI (main.py)       |
|  - Текстовое меню программы              |
|  - Сбор ввода и форматированный вывод    |
+------------------------------------------+
                     │
                     ▼
+------------------------------------------+
|  Слой бизнес-логики (converter.py)      |
|  - Валидация валют, сумм и курсов        |
|  - Расчет конвертации и обработка ошибок |
+------------------------------------------+
                     │
                     ▼
+------------------------------------------+
|  Слой данных / БД (database.py)          |
|  - Инициализация таблиц SQLite           |
|  - CRUD-операции с курсами и историей    |
+------------------------------------------+
                     │
                     ▼
+------------------------------------------+
|    Файл базы данных (currency_converter.db)|
+------------------------------------------+ 

## 4. Инструкция по установке и запуску
Требования
Установленный интерпретатор Python 3.8 или выше.

Менеджер пакетов pip (поставляется вместе с Python).

Установка зависимостей
Для запуска тестов и генерации отчетов о покрытии кода установите необходимые пакеты:

- pip install pytest pytest-cov
Запуск приложения
Запустите главный файл интерфейса из корневой директории проекта через терминал:

python main.py

---

## 5. Запуск тестов и проверка покрытия (TDD)
Бизнес-логика приложения полностью изолирована. В процессе автоматического тестирования используется временная база данных, что гарантирует высокую скорость выполнения тестов и чистоту ваших локальных файлов.

Запуск всех тестов в подробном режиме:

- python -m pytest -v
Запуск тестов с анализом покрытия кода (Code Coverage):

- python -m pytest --cov=converter --cov=database tests/
Важно: Согласно критериям оценки курсового проекта, минимально допустимый уровень покрытия кода тестами (Total Coverage) составляет 80%. Наше приложение покрыто тестами на 100%.

## 6. Часто задаваемые вопросы (FAQ)
Вопрос: Где физически хранятся данные программы и как сделать их резервную копию (бэкап)?

Ответ: Все курсы валют и история операций записываются в файл currency_converter.db, который автоматически создается в корневой папке проекта при первом запуске приложения. Чтобы сделать бэкап или перенести данные на другой компьютер, достаточно просто скопировать этот файл.
